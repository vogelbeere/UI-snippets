Quick accessibility checklist

What Who benefits  Why Solution More info 
Alt attribute All (especially visually impaired and slow modem users)  W3C guideline: alt attribute must always be present to provide alternative text if images are switched off 

Use title if you want a tooltip to appear; don't use it if you don't want a tooltip 		Always use alt attribute; use title attribute for illustrations  Web Content Accessibility Guidelines (WCAG) 2.0
Colour contrast  Visually impaired / colour blind / all  It's difficult to read dark text on a dark background  Use colours that contrast effectively  Paletton.com 
Don't use colour alone to convey meaning Colour blind people; blind people  A tick in a green circle and a cross in a red circle are OK because they also have a tick and a cross in them; if they were just a red circle and a green circle, they wouldn't be sufficiently different Use semantic markup, default link styles, and more than one way of conveying meaning  Use colour safely 
If you specify text colour in CSS, must also specify background colour  Users of alternate browser settings If you have specified black text but the user has specified a black background, they won't be able to see the text unless you have also specified an appropriate background colour     
JavaScript Slow modem / Lynx users  If these users have JavaScript turned off, will all your page functionality work?  Progressive enhancement  Progressive enhancement
Label form elements  All (especially people with motor impairments such as RSI)  If you label form elements, it provides a larger area for the user to click on  Label form elements  Labelling form elements 
Linearised tables  Users of screen-readers; all  Text in your table must read from left-to-right for the screen reader to interpret it correctly  Linearise your tables  
Meaningful link titles  Users of screen-readers; all Because if someone chooses the option to only have link titles read to them by their screen reader, then "click here, click here, click here" will not be very useful. Also, "click here" is naff and unprofessional; and don't use meaningless titles like "this document" as a substitute  Think before you link  W3C guideline on linking 
Offer access keys  All (especially people with motor impairments such as RSI) If you can't use a mouse or don't want to, you can press Alt + [accesskey] to get the functionality you want; also mimics Windows shortcuts.  Use access keys (which should be defined in accessibility statement)  Defining keyboard shortcuts 
Separate styling from content  All; users of alternate browser settings  If people have set their own colour preferences in the browser, don't override them by having styling in the page.  Use HTML 5 and CSS 3. Put scripts and styles in separate files.
